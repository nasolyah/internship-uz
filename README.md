# internship.uz

Платформа стажировок для стартапов и студентов Узбекистана (пилот). Одностраничное приложение
на ванильном JavaScript + Supabase для аутентификации.

## Структура

| Файл | Назначение |
|------|-----------|
| `index.html` | Точка входа, подключает Supabase JS и приложение |
| `int_app.js` | Всё приложение (SPA): вьюхи, состояние, аутентификация |
| `int_css.css` | Стили и анимации |
| `supabase/functions/telegram-auth/` | Edge Function: проверяет подпись Telegram и выдаёт Supabase-сессию |
| `supabase/migrations/` | Схема БД и политики RLS |
| `SETUP_TELEGRAM.md` | Инструкция по настройке входа через Telegram |

## Аутентификация

**Студенты** — два способа входа:

- **Telegram** — своя кнопка вызывает `Telegram.Login.auth`, данные проверяются в Edge
  `telegram-auth` (HMAC-SHA256 токеном бота), затем клиент обменивает OTP на сессию через
  `verifyOtp`. Подробности — в [SETUP_TELEGRAM.md](SETUP_TELEGRAM.md).
- **Email** — одноразовый код через `signInWithOtp` / `verifyOtp`.

**Компании** — одноразовый код на корпоративную почту (тот же `signInWithOtp`). Заявка
компании привязана к аккаунту через `company_applications.owner_user_id`; публиковать задачи
может только заявка со статусом `approved`.

Секреты (токен бота, service-role ключ) живут только в секретах Supabase Edge Functions —
в клиентском коде их нет. Публичный `anon`-ключ в `int_app.js` открыт по дизайну.

## Отклики и чат

Студент откликается на задачу — в `gig_applications` появляется строка, а триггер тут же
создаёт в `messages` системное сообщение. Отклик и есть ветка чата: студент видит её в
«Моих откликах», компания — в «Моих вакансиях» под соответствующей задачей. Новые сообщения
приходят через Supabase Realtime.

Обе стороны — обычные пользователи `auth.users`, поэтому доступ к переписке разрешает RLS:
читать и писать в ветку может только её участник.

## Локальный запуск

Статика — достаточно любого статик-сервера:

```bash
python -m http.server 8777
# затем открыть http://localhost:8777/index.html
```

> Вход через Telegram работает только на публичном домене, привязанном к боту через
> BotFather `/setdomain` (у нас — `internship-uz.pages.dev`): на `file://` и `localhost`
> кнопка авторизации не сработает. Вход по коду на почту работает и локально — им и
> проверяются отклики с чатом.

## Деплой

Прод: **https://internship-uz.pages.dev** — статика на Cloudflare Pages, собирается
автоматически при push в `main`. То есть push и есть деплой сайта.

Порядок при изменениях, затрагивающих базу: сначала миграции, потом Edge Functions,
и только затем push — иначе новый клиент какое-то время будет ходить в схему, которой
ещё нет.

```bash
supabase db push                                   # миграции
supabase functions deploy <имя>                    # или через Supabase Dashboard
git push origin main                               # сайт (Cloudflare пересоберёт)
```

`tg-webhook` деплоится с `--no-verify-jwt` — Telegram не присылает наш JWT, эндпоинт
защищён секретом в заголовке.
